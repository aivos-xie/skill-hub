## 一、基础概念

### 1.1 Git是什么?

Git是一个分布式版本控制系统,用于跟踪文件变化、协作开发和代码管理。

**核心概念**:
- **Repository (仓库)**: 存储项目文件和历史的目录
- **Commit (提交)**: 一次快照,记录文件的某个状态
- **Branch (分支)**: 独立的开发线路
- **Remote (远程)**: 托管在服务器上的仓库副本

### 1.2 工作流程

```
工作区 → 暂存区 → 本地仓库 → 远程仓库
(Working) (Staging) (Local)  (Remote)
```

## 二、初始化与配置

### 2.1 创建仓库

```bash
# 初始化新仓库
git init

# 克隆远程仓库
git clone https://github.com/user/repo.git
git clone git@github.com:user/repo.git  # SSH方式
```

### 2.2 用户配置

```bash
# 设置用户名和邮箱 (全局)
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

# 查看配置
git config --list
git config user.name

# 设置默认编辑器
git config --global core.editor "vim"

# 设置换行符处理
git config --global core.autocrlf true   # Windows
git config --global core.autocrlf input  # macOS/Linux
```

## 三、基本操作

### 3.1 查看状态

```bash
# 查看工作区状态
git status

# 查看详细变化
git status -v

# 简短状态
git status -s
```

输出示例:
```
M  modified_file.txt    # 已修改
A  new_file.txt         # 已添加到暂存区
D  deleted_file.txt     # 已删除
?? untracked_file.txt   # 未跟踪
```

### 3.2 添加文件

```bash
# 添加指定文件到暂存区
git add file.txt

# 添加所有变化
git add .
git add -A

# 交互式添加
git add -i
```

### 3.3 提交更改

```bash
# 提交暂存区的更改
git commit -m "Add new feature"

# 提交并编辑详细信息
git commit

# 修改上次提交
git commit --amend
git commit --amend -m "Updated message"

# 跳过暂存区直接提交 (已跟踪文件)
git commit -a -m "Quick commit"
```

### 3.4 查看历史

```bash
# 查看提交历史
git log
git log --oneline          # 单行显示
git log --graph            # 图形化分支
git log --all --decorate   # 显示所有分支

# 查看最近N次提交
git log -n 5

# 查看某文件的修改历史
git log --follow file.txt

# 查看具体变化
git show <commit-hash>
git show HEAD
```

## 四、分支管理

### 4.1 分支操作

```bash
# 列出分支
git branch           # 本地分支
git branch -r        # 远程分支
git branch -a        # 所有分支

# 创建分支
git branch feature-login
git checkout -b feature-login  # 创建并切换

# 切换分支
git checkout main
git switch main      # Git 2.23+ 推荐

# 删除分支
git branch -d feature-login   # 已合并
git branch -D feature-login   # 强制删除
```

### 4.2 合并分支

```bash
# 切换到目标分支
git checkout main

# 合并feature分支
git merge feature-login

# 合并策略
git merge --no-ff feature-login    # 禁用快进
git merge --squash feature-login   # 压缩提交

# 解决冲突后继续
git add resolved_file.txt
git commit
```

### 4.3 变基 (Rebase)

```bash
# 变基到main
git checkout feature
git rebase main

# 交互式变基 (修改历史)
git rebase -i HEAD~3

# 中止变基
git rebase --abort

# 继续变基 (解决冲突后)
git rebase --continue
```

## 五、远程仓库

### 5.1 远程操作

```bash
# 查看远程仓库
git remote -v

# 添加远程仓库
git remote add origin https://github.com/user/repo.git

# 重命名远程
git remote rename origin upstream

# 删除远程
git remote remove origin
```

### 5.2 推送和拉取

```bash
# 推送到远程
git push origin main
git push -u origin main  # 设置上游分支

# 强制推送 (谨慎使用!)
git push --force
git push --force-with-lease  # 更安全

# 拉取远程更新
git pull origin main
git pull --rebase            # 变基方式拉取

# 仅获取不合并
git fetch origin
git fetch --all
```

### 5.3 同步 fork

```bash
# 添加上游仓库
git remote add upstream https://github.com/original/repo.git

# 获取上游更新
git fetch upstream

# 合并到本地
git checkout main
git merge upstream/main

# 推送到自己的fork
git push origin main
```

## 六、暂存 (Stash)

```bash
# 暂存当前工作
git stash
git stash save "WIP: login feature"

# 查看暂存列表
git stash list

# 应用暂存
git stash apply          # 应用但不删除
git stash pop            # 应用并删除

# 应用指定暂存
git stash apply stash@{2}

# 删除暂存
git stash drop stash@{0}
git stash clear          # 清空所有

# 从暂存创建分支
git stash branch new-branch stash@{0}
```

## 七、撤销操作

### 7.1 撤销工作区修改

```bash
# 撤销文件修改
git checkout -- file.txt
git restore file.txt     # Git 2.23+

# 撤销所有未暂存的修改
git checkout -- .
```

### 7.2 撤销暂存

```bash
# 从暂存区移除
git reset HEAD file.txt
git restore --staged file.txt  # Git 2.23+

# 撤销所有暂存
git reset HEAD
```

### 7.3 撤销提交

```bash
# 软重置 (保留变化和提交信息)
git reset --soft HEAD~1

# 混合重置 (保留变化,清除提交)
git reset HEAD~1

# 硬重置 (丢弃所有变化!)
git reset --hard HEAD~1

# 恢复已删除的提交
git reflog
git reset --hard <commit-hash>
```

### 7.4 Revert (安全的撤销)

```bash
# 创建新的提交来撤销
git revert <commit-hash>
git revert HEAD
git revert HEAD~2..HEAD  # 撤销多个
```

## 九、高级技巧

### 9.1 Cherry-pick

```bash
# 选取特定提交应用到当前分支
git cherry-pick <commit-hash>
git cherry-pick <hash1> <hash2>
git cherry-pick <hash1>..<hash2>

# 有冲突时
git cherry-pick --continue
git cherry-pick --abort
```

### 9.2 Bisect (二分查找bug)

```bash
# 开始bisect
git bisect start
git bisect bad    # 当前版本有问题
git bisect good v1.0.0  # 这个版本没问题

# Git会自动checkout中间版本,测试后标记:
git bisect good   # 或 git bisect bad

# 找到问题提交后重置
git bisect reset
```

### 9.3 Submodule

```bash
# 添加子模块
git submodule add https://github.com/user/lib.git

# 初始化子模块
git submodule init
git submodule update

# 克隆含子模块的仓库
git clone --recursive https://github.com/user/repo.git

# 更新子模块
git submodule update --remote
```

### 9.4 Worktree

```bash
# 创建工作树 (同时处理多个分支)
git worktree add ../feature-branch feature/login
git worktree list
git worktree remove ../feature-branch
```

## 十一、最佳实践

### 11.1 提交规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

Type类型:
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具

示例:
```
feat(auth): add login validation

- Add email format check
- Add password strength requirement

Closes #123
```

### 11.2 分支策略

```
main (生产)
├── develop (开发)
│   ├── feature/login
│   ├── feature/payment
│   └── hotfix/bug-fix
└── release/v1.0
```

### 11.3 安全提示

- ⚠️ 永远不要提交密码、密钥、token
- ⚠️ 使用 `.gitignore` 排除敏感文件
- ⚠️ 定期更新Git以修复安全漏洞
- ⚠️ 使用SSH密钥而非HTTPS密码
