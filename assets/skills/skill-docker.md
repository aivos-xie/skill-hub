## 一、核心概念

**镜像 (Image):** 只读模板,用于创建容器。
**容器 (Container):** 镜像的运行实例,隔离进程。
**仓库 (Registry):** 存储镜像的中央服务(Docker Hub)。
**Dockerfile:** 定义镜像构建步骤的文本文件。
**Compose:** 定义多容器应用的工具。

## 二、镜像管理

```bash

# 拉取镜像


docker pull nginx
docker pull python:3.11-slim

# 查看镜像


docker images
docker images --format "table {{.Repository}}\t{{.Tag}}\t{{.Size}}"

# 构建镜像


docker build -t myapp:1.0 .
docker build --no-cache .
docker build --build-arg VERSION=1.0 .

# 打标签与推送


docker tag myapp:1.0 user/myapp:1.0
docker push user/myapp:1.0

# 删除镜像


docker rmi image_id
docker rmi -f image_id
docker image prune
docker image prune -a
```

## 六、Docker Compose

```yaml
version: '3.8'
services:
  web:
    build: .
    image: myapp:1.0
    ports:

      - "8080:8080"
    environment:

      - APP_ENV=production
    depends_on:

      - db
    volumes:

      - ./data:/app/data
    restart: unless-stopped
  db:
    image: postgres:15
      POSTGRES_PASSWORD: secret

      - pgdata:/var/lib/postgresql/data
  pgdata:

docker compose up -d                  # 后台启动
docker compose down                   # 停止并删除
docker compose ps                     # 查看状态
docker compose logs -f web            # 查看日志
docker compose exec web /bin/bash     # 进入容器
docker compose build                  # 构建镜像
docker compose config                 # 验证配置

## 七、Dockerfile 指令速查

```dockerfile
FROM python:3.11-slim                # 基础镜像
WORKDIR /app                          # 工作目录
ENV APP_ENV=production                # 环境变量
ARG BUILD_VERSION=dev                 # 构建参数
COPY requirements.txt .               # 复制文件
RUN pip install -r requirements.txt   # 执行命令
EXPOSE 8080                           # 暴露端口
USER appuser                          # 切换用户
CMD ["python", "app.py"]              # 启动命令
HEALTHCHECK CMD curl -f http://localhost:8080/ || exit 1

## 八、最佳实践

- 使用官方基础镜像
- 选择轻量镜像(alpine/slim)
- 多阶段构建减小最终镜像体积
- 合并 RUN 指令减少层数
- 使用 .dockerignore 排除无关文件
- 以非root用户运行
- 添加 HEALTHCHECK 健康检查
- 明确 WORKDIR
- 扫描漏洞: docker scan myimage
