## 一、操作系统 (Operating System)

### 1.1 进程与线程

**进程 (Process)**: 程序执行的实例,资源分配的最小单位。
- 包含代码、数据、PCB (Process Control Block)
- 拥有独立的内存空间

**线程 (Thread)**: CPU调度的最小单位。
- 同一进程内的线程共享内存空间
- 上下文切换开销比进程小

**进程状态**:
```
新建(NEW) -> 就绪(READY) -> 运行(RUNNING) -> 等待(WAITING) -> 终止(TERMINATED)

**进程间通信 (IPC)**:
- 管道 (Pipe): 半双工,父子进程间
- 命名管道 (FIFO): 任意进程
- 消息队列 (Message Queue)
- 共享内存 + 信号量
- 套接字 (Socket): 跨主机
- 信号 (Signal)

### 1.2 CPU调度

**调度目标**: CPU利用率最大化;吞吐量最大化;周转/等待/响应时间最小化。

**常见调度算法**:

| 算法         | 描述                    | 特点                     |
|-------------|-------------------------|--------------------------|
| FCFS        | 先到先服务               | 简单,公平                |
| SJF / SRTF  | 短作业优先/抢占式短作业   | 最优平均等待时间         |
| 优先级调度   | 高优先级先执行           | 可能饥饿,可用老化解决    |
| Round Robin | 时间片轮转               | 公平,响应好,分时系统     |
| 多级队列    | 多个就绪队列             | 前台/后台分离            |
| 多级反馈队列 | 可在队列间迁移           | 综合性能好,应用广泛      |

### 1.3 内存管理

**内存管理功能**: 地址重定位;内存分配与回收;内存保护;内存扩充(虚拟内存)。

**存储分配方式**:
1. **连续分配**: 单一分区 / 固定分区 / 动态分区(首次适应/最佳适应/最坏适应)
2. **分页存储 (Paging)**: 逻辑空间分页,物理空间分页框;页表记录映射;快表(TLB)加速
3. **分段存储 (Segmentation)**: 按逻辑单位分段;符合用户视角
4. **段页式**: 先分段,段内分页

### 1.4 虚拟内存

**核心思想**: 进程不必全部装入内存即可运行。
**局部性原理**: 时间局部性(最近访问的很可能再次访问);空间局部性(相邻地址很可能被访问)。

**页面置换算法**:

| 算法         | 思想                    | 特点                     |
| FIFO        | 最早进入的先淘汰         | 实现简单,Belady异常      |
| LRU         | 最久未使用的先淘汰       | 性能好,近似最优          |
| OPT/Min     | 未来最久不使用的先淘汰   | 理论最优,不可实现(参考)  |
| Clock       | 时钟算法(访问位)         | LRU近似,低开销           |
| 工作集模型  | 保留进程当前工作集页面   | 防止抖动(thrashing)      |

### 1.5 文件系统与I/O

**文件组织**: 顺序文件;索引文件;索引顺序文件;直接/散列文件。

**空闲空间管理**: 位示图(Bitmap);空闲链表;成组链接。

**I/O控制方式**: 程序查询(轮询);中断驱动;DMA;通道方式。

**磁盘调度**: FCFS;SSTF(最短寻道时间优先);SCAN(电梯算法);C-SCAN(循环扫描);LOOK/C-LOOK。

### 1.6 死锁 (Deadlock)

**四个必要条件 (Coffman条件)**:
1. **互斥 (Mutual Exclusion)**: 资源独占使用
2. **占有并等待 (Hold and Wait)**: 持有资源并等待其他资源
3. **不抢占 (No Preemption)**: 资源不能被强行剥夺
4. **循环等待 (Circular Wait)**: 形成环状资源等待链

**处理策略**:

| 策略         | 方法                    | 例子                     |
| 死锁预防    | 破坏任一必要条件         | 一次性申请所有资源        |
| 死锁避免    | 安全状态检查             | 银行家算法(Banker's)     |
| 死锁检测    | 检测并报告               | 资源分配图(RAG)检测环    |
| 死锁解除    | 检测后恢复               | 撤销进程/抢占资源        |
| 忽略        | 鸵鸟算法                 | 多数桌面系统             |
## 二、计算机网络 (Computer Networks)

### 2.1 OSI 与 TCP/IP 模型

应用层 / 表示层 / 会话层 -> 应用层 (HTTP, DNS, FTP)
传输层 -> 传输层 (TCP, UDP)
网络层 -> 网络层 (IP, ICMP, ARP)
数据链路层 -> 链路层 (Ethernet)
物理层 -> 物理层 (电缆/无线)

### 2.2 TCP vs UDP

| 特性         | TCP                     | UDP                      |
| 连接性      | 面向连接(三次握手)       | 无连接                   |
| 可靠性      | 可靠(确认+重传)          | 不可靠                   |
| 顺序性      | 保序                    | 不保序                   |
| 流量控制    | 滑动窗口                 | 无                       |
| 拥塞控制    | 有                      | 无                       |
| 速度        | 较慢                    | 快                      |
| 适用场景    | HTTP/SSH/邮件            | 视频/语音/游戏/DNS        |

### 2.3 TCP 三次握手与四次挥手

**三次握手**:
Client -> SYN(seq=x) -> Server
Client <- SYN+ACK(seq=y,ack=x+1) <- Server
Client -> ACK(seq=x+1,ack=y+1) -> Server (ESTABLISHED)

**四次挥手**:
Client -> FIN -> Server (CLOSE_WAIT)
Client <- ACK <- Server
Client <- FIN <- Server
Client -> ACK -> Server (TIME_WAIT 2MSL)

**TIME_WAIT 作用**: 保证最后一个ACK到达对方;让老的重复分节在网络中消失。

### 2.4 TCP 流量控制与拥塞控制

**流量控制**: 接收方通过TCP头部窗口大小(Window)字段告知发送方;滑动窗口协议。

**拥塞控制四个核心算法**:
1. **慢启动 (Slow Start)**: cwnd指数增长
2. **拥塞避免 (Congestion Avoidance)**: 达到ssthresh后线性增长
3. **快重传 (Fast Retransmit)**: 收到3个重复ACK立即重传
4. **快恢复 (Fast Recovery)**: cwnd减半而非回到1

### 2.5 HTTP 与 HTTPS

**HTTP 常见方法**: GET(获取资源);POST(提交数据);PUT(更新资源,幂等);DELETE(删除资源);HEAD(仅响应头);OPTIONS(支持方法);PATCH(部分更新)。

**HTTP 状态码**: 1xx信息(100 Continue);2xx成功(200 OK,201 Created,204 No Content);3xx重定向(301/302/304);4xx客户端错误(400/401/403/404);5xx服务器错误(500/502/503/504)。

**HTTP vs HTTPS**: HTTP明文(端口80);HTTPS TLS加密(端口443)。

**HTTP/2 改进**: 多路复用(单TCP连接多请求);头部压缩(HPACK);服务器推送;二进制协议。

**HTTP/3**: 基于QUIC(UDP),更快连接建立,无队头阻塞。

### 2.6 DNS (Domain Name System)

**层级结构**: 根域名服务器 -> TLD(.com/.org) -> 权威服务器 -> 递归/本地DNS服务器。

**解析过程**: 浏览器缓存 -> 系统缓存 -> hosts文件 -> 本地DNS服务器(递归) -> 根->TLD->权威(迭代)。

**DNS记录类型**: A(IPv4);AAAA(IPv6);CNAME(别名);MX(邮件);NS(域名服务器);TXT(文本信息/SPF/DKIM);PTR(反向解析)。

### 2.7 路由与转发

| 算法          | 类型       | 代表协议 |
|--------------|------------|----------|
| 距离向量(DV) | 分布式    | RIP      |
| 链路状态(LS) | 分布式    | OSPF     |
| 路径向量     | 分布式    | BGP      |

- **RIP**: 跳数为度量,最大跳数15,定期广播路由表
- **OSPF**: 链路状态协议,Dijkstra算法,仅泛洪链路变化,支持区域划分
- **BGP**: 路径向量协议,自治系统间路由,策略丰富,互联网骨架协议

### 2.8 ARP & ICMP

**ARP (Address Resolution Protocol)**: IP地址 -> MAC地址;广播请求,单播响应;ARP缓存;免费ARP(检测IP冲突);ARP欺骗攻击。

**ICMP (Internet Control Message Protocol)**: Echo Request/Reply(ping);Destination Unreachable;Time Exceeded(TTL=0,tracert原理);Redirect;Parameter Problem。

### 2.9 网络安全

**常见攻击**: DoS/DDoS(拒绝服务);SYN洪水(利用TCP三次握手漏洞);MITM中间人攻击(HTTPS防御);SQL注入;XSS跨站脚本;CSRF跨站请求伪造。

**TLS/SSL 握手简化**:
1. ClientHello: 支持的密码套件,随机数
2. ServerHello: 选定套件,证书,随机数
3. 客户端验证证书,生成预主密钥(PMS)
4. 使用PMS+随机数派生会话密钥
5. 双方切换加密通信
